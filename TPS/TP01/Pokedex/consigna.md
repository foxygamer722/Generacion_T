## Caso elegido 1: Pokédex con favoritos

<br>

### Prompt original
"Haceme una pagina de Pokémon con favoritos."
<br>

### Prompt mejorado
"Haceme una pagina hecha con HTML/CSS/JS de pokemon con estilo retro, utilizando PokeAPI, que utilice localStorage, manejo de errores, diseño responsive y estructura de componentes, con sistema de favoritos."
<br>

### Qué información agregué
Agregué varios elementos al prompt original siendo:
- **LENGUAJE:** especifique con que lenguajes debia trabajar la pagina.
- **DISEÑO:** se le agrego al prompt que tenia que tener estilo retro ademas con un diseño responsive.
- **API:** si bien fue utilizado en el prompt original, pudo ser el caso de que no lo hiciera o usara otro el cual tiene menos informacion.
- **LOCALSTORAGE:** se le especifico que utilizara localStorage para almacenar la informacion de forma local.
- **SISTEMA:** se le agrego el manejo de errores y el sistema de favoritos.
<br>

### Riesgos del prompt original
Si bien quedo bien el prompt original, al ser tan general y sin ninguna especificacion puede surgir errores o generar cosas que no deseamos, cosa que en el prompt mejorado no ocurre por ejemplo: con el diseño y el localStorage.

### Validación personal
El codigo es entendible ya que esta dividido en bloques aunque en ambos prompts son un solo archivo html siendo complicado encontrar algun error, en vez de tener separado el html,css y el javascript.
Obviamente validaria de que no haya ningun error (codigo, api e visual) ademas de que se le indico esta aplicado y de forma correcta.
