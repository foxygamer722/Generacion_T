## Caso elegido 3: App Star Wars para estudiantes.

<br>

### Prompt original
"Haceme una página de Star Wars."
<br>

### Prompt mejorado
"Haceme una página de Star Wars con HTML/CSS/JS o react que actue como una wiki interactiva, API Publica utilizando esta para la informacion y ampliar con informacion de la wiki de star wars, que explique a detalle con imagenes (personajes,planetas,naves,especies,organizaciones,peliculas,series, star wars legends y el cannon),que tenga diseño inspirado en la ciencia ficcion, con pantalla de carga y en caso de error que muestre el error, manejo de carga y error."
<br>

### Qué información agregué
Agregué varios elementos al prompt original siendo:
- **LENGUAJE:** especifique con que lenguajes debia trabajar la pagina.
- **DISEÑO:** se le agrego al prompt que la pagina tenia que ser una wiki interactiva y tener estilo inspirado en la ciencia ficcion ademas de una pantalla de carga e imagenes.
- **API:** se le agrego al prompt mejorado ya que en el prompt original no uso la SWAPI perdiendo informacion.
- **SISTEMA:** se le agrego el manejo de errores, que muestre si ocurre un error.
- **INFORMACION:** se le especifico que informacion se quiere agregar a la wiki.
<br>

### Riesgos del prompt original
El prompt original hizo una pagina de Star Wars simple con un diseño generico, con un apartado de la fuerza que no aporta nada, sin absolutamente nada de informacion mientras que en el prompt mejorado al agregarse estas especificaciones se solucionan estos problemas quedando una pagina con un diseño original, con cosas utiles y completas de informacion.

### Validación personal
El codigo es entendible ya que esta dividido en bloques aunque en el prompt original es un solo archivo html siendo complicado encontrar algun error y luego el prompt mejorado quedo separado con el .hmtl y el .jsx.
Antes de usarlo validaria, si funciona todo correctamente que en el caso del prompt mejorado se puede ver que las imagenes no aparecen o no son las correspondientes, siendo confuso.
